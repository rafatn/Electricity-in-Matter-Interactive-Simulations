Integrated Antiviral Model

This document describes the fully integrated concept: a 2D airborne spread engine, binary / hex state encoding (including air-risk bits), and a treatment graph with DFS + Dijkstra, all wired together in a single Python module.

Airborne spread Binary / Hex state DFS + Dijkstra Fair treatment logic

1. From Room Physics to State Bits

The integrated Python module integrated_virus_model.py first simulates the aerosol-based virus concentration in a room using a 2D grid model. From the summary (max and average concentration), it classifies the airborne risk into three levels: low, medium, and high.

These risk classes are encoded as extra bits in the binary state vector:

bit 16 → air_risk_medium = 1
bit 17 → air_risk_high = 1

The final state ID is a hexadecimal string like 0xA3F21, combining virus properties, patient comorbidities, room conditions, and airborne risk into a single compact signature.

2. Treatment Graph Construction

Using these states, the module builds a decision graph:

  • S₀ – baseline room, no UVC, poor ventilation, high air risk, unstable patient.
  • S₁ – UVC activated, air risk reduced, patient still unstable.
  • S₂ – UVC + improved ventilation, air risk further reduced.
  • S₃ – UVC + ventilation + pharmacologic support, patient stabilized.

Edges as Treatment Actions

Each edge in the graph represents a clinical or environmental action, such as “Activate UVC lighting” or “Optimize room ventilation”. Each edge has an abstract cost combining time, risk, and resource usage.

Air-Risk in the Path

Because the air-risk bits are part of the state, the shortest path must also reduce airborne concentration over time. This links physical reality in the room directly to the optimization logic.

3. Python Demo Usage

In the module, the demo() function runs the complete pipeline: simulate three room configurations (baseline, UVC, UVC+ventilation), classify air-risk, build the graph, run DFS, and then Dijkstra. 

from integrated_virus_model import demo

if __name__ == "__main__":
    demo()

The console output shows: airborne risk metrics, a DFS exploration of the treatment branches, and the optimal path from baseline high-risk to stable low-risk state.
